Below is the complete data model that you'll receive from the iframe messages.

**Please note** - we have additional fields available that we are not currently sending. If you're application needs more data that we are not currently sending, please contact us.

```typescript
interface RoleplayResult {
    // Unique identifier for the roleplay session
    id: string; // UUID

    // Performance metrics
    length: number; // Duration of the roleplay in milliseconds
    wpm: number; // Words per minute
    name_count: number; // Number of times names were mentioned
    num_questions: number; // Number of questions asked
    num_filler_words: number; // Number of filler words used
    score: number; // Overall score (0-100)

    // Timestamps
    created_at: Date; // ISO timestamp - roughly equates to when the roleplay start time
    updated_at: Date; // ISO timestamp - roughly equates to when the roleplay end time

    // Activity information
    activityId: string; // UUID
    activityType: "premade-roleplay" | "script" | "start-screen";

    // User information - These are the claims you pass us through the JWT
    user: {
        learnerName: string;
        originDomain: string;
        uniqueIdentifier: string;
    };

    // Roleplay configuration
    premade_roleplay: {
        title: string;
        passing_score: number;
    };

    // Scenario details
    scenario: {
        title: string;
    };

    // Customer information
    customer_persona: {
        image_url: string;
        name: string;
    };

    customer_situation: {
        job_title: string;
        company_name: string;
    };

    // Conversation transcript
    snippets: Array<{
        is_user: boolean;
        text: string;
    }>;

    // Evaluation responses
    prompt_responses: Array<{
        title: string;
        passed: boolean;
        explanation: string | null;
    }>;
}
```
